Objectif de cette section

L’un des facteurs de succès de Python réside dans ses bibliothèques spécialisées.

Vocabulaire

Bibliothèque
Une bibliothèque est une collection de packages et de modules. Par exemple la bibliothèque Numpy a pour package principal numpy.

En Python, il existe deux types de structure pour organiser votre code: les packages et les modules.

Package
Un package d’une bibliothèque est un dossier contenant un ou plusieurs modules et des sous-packages organisés hiérarchiquement. Il facilite la programmation modulaire en permettant de découper une tâche de programmation complexe en sous-tâches plus gérables.Source [https://realpython.com/python-modules-packages/ ]

Exemple de structure d’un package:

mypackage/
├── init.py
├── module1.py
├── module2.py
└── …
Par exemple, le package numpy de la bibliothèque Numpy contient les modules numpy.core, numpy.linalg…
Autre exemple, le package de calcul scientifique scipy de la bibliothèque Scipy contient le module stats de calcul statistique qui contient lui-même la fonction chi2_contingency() qui permet d’effectuer un test du chi2.
Module
Le terme module désigne un fichier Python dans lequel sont stockés des fonctions, des classes des sous-modules et des variables. Cela peut être utile dans un contexte général, car cela permet de clarifier le type d’objet dont on parle.
En pratique
Les termes bibliothèques, package et modules sont souvent utilisés de manière interchangeable. Par exemple, on pourra dire “le module NumPy” ou “la bibliothèque NumPy”.

Les packages sont un moyen puissant d’organiser et de structurer votre code Python, ce qui facilite la gestion de projets complexes.

Créer un package (avancé)

Quand les modules sont trop consistants ou que l’on souhaite les publier dans le PyPi ou GitHub il faut alors passer à la notion de package (ensemble de fichiers .py rassemblés et ordonnés dans un répertoire).

Le fichier d’initialisation du package init.py est placé à la racine.
Au chargement du package, Python lance le code qui se trouve dans le fichier. Ce fichier peut être vide. On peut y ajouter la ligne de code ‘from mypackage.mon_code import *’ cela permettra le préchargement des fichiers se trouvant dans le répertoire (oups le package :))

Remarque: le nom du package est en minuscules et il faut éviter de lui donner un nom existant.

mypackage/
├── init.py
├── module1.py
├── module2.py
└── …

Tableaux de nombres tirés aléatoirement

Pour générer des tableaux plus complexes, on peut utiliser le module np.random de numpy.
Ce module dispose sous-modules générateurs de nombre aléatoire parmi lesquels:

  • Loi discrète: np.random.randint génère des nombres entiers tirés aléatoirement entre des bornes à définir$.

  • Loi uniforme: np.random.rand génère des nombres tirés uniformément dans \([0, 1]\).

  • Loi normale: np.random.randn génère des nombres tirés selon une loi normale \(\mathcal N(0, 1)\).

  • Loi normale: np.random.randn génère des nombres tirés selon une loi normale \(\mathcal N(loc, scale, size)\).

Il expose aussi des fonctions de plus haut niveau pour différentes lois de probabilité.

La fonction np.random.randint(low, high, size) du sous-module np.random de Python renvoie un nombre entier aléatoire compris entre deux bornes

La fonction prend trois arguments :

  • low (optionnel): la borne inférieure de la plage de nombres aléatoires qui est incluse. Si elle n’est pas spécifiée sa valeur par défaut sera 0.

  • high : la borne supérieure de la plage de nombres aléatoires qui est excluse.

  • size (optionnel): la taille de l’échantillon. Par ex. size=(4,3). Si size n’est pas spécifiée, par défaut l’échantillon est réduit à une seule valeur.
